# 만세력 · 사주 계산 엔진 (saju)

> 생년월일시 + 성별 → **사주 원국·십성·지장간·12운성·12신살·신강약·용신·대운·세운·일진**을
> **결정론적으로 계산**하는 만세력 엔진 + 그 결과를 보여주는 웹앱.
>
> **라이브: https://inno-hi-inc.github.io/saju/**

`AI`가 사주를 "지어내는" 서비스가 아니다. **계산은 오차 0의 결정론 엔진**이 하고,
해석(성향·운세 서술)만 나중에 LLM이 맡는 구조다. (아래 [핵심 설계 원칙](#핵심-설계-원칙) 참고)

## 📁 저장소 구조

이 저장소 **하나에 웹앱과 계산 엔진이 모두** 들어있다.

```
saju/                        ← 저장소 루트 = GitHub Pages 루트
├── index.html               웹앱 진입점 (Pages가 이걸 서빙)
├── css/style.css            토스 테마 플랫 디자인
├── js/
│   ├── saju-engine.js       ★ 만세력 엔진 (브라우저 JS) — 라이브에서 실제로 도는 코드
│   ├── app.js               UI 렌더링 (폼·화면전환·이미지저장·저장/불러오기)
│   └── lunar-table.json     음↔양력 변환표 (1901~2099, KASI 교차검증)
│
├── engine/                  ★ 같은 만세력 로직의 Python 원본 (검증·상용화 백엔드용)
│   ├── saju/                결정론 엔진 패키지 (constants·solarterms·pillars·derive·strength·luck·chart)
│   ├── tests/               픽스처 검증 (1998-06-08 = 31/31 통과)
│   ├── demo.py              콘솔 렌더 / JSON 출력
│   ├── app.py               Streamlit 데모
│   └── README.md            엔진 상세 문서
│
├── README.md                ← 지금 이 문서
├── HANDOVER.md              ★ 인수인계 체크리스트 (계정·배포·로드맵·함정) — 먼저 읽을 것
└── DEPLOY.md                배포 방법
```

> **왜 엔진이 두 벌(JS + Python)인가?**
> - `js/saju-engine.js` = **지금 라이브에서 실제로 도는 코드**. 서버 없이 방문자 브라우저에서 계산.
> - `engine/`(Python) = **동일 로직의 검증용 원본** + 상용화 시 서버 백엔드로 쓸 코드.
> - ⚠️ **둘은 반드시 같은 값을 내야 한다.** 한쪽을 고치면 다른 쪽도 고쳐야 함 → [엔진 동기화](#-엔진-동기화-js--python) 참고.

## 핵심 설계 원칙

**계산과 해석의 분리.** 상용화의 핵심이라 절대 흐트러뜨리지 말 것.

```
[이 엔진]  결정론 계산 (오차 0)                 [LLM 레이어]  서술만
 8글자·십성·지장간·12운성·12신살    ──JSON 계약──▶   성향·운세 해석
 신강약·용신·대운·세운·일진                        (지금:없음 → 나중:클라우드 API)
```

- **만세력은 절대 LLM에게 계산시키지 않는다.** LLM은 `buildChart()`가 반환한 JSON을 *읽어서 서술*만 한다.
- 해석 LLM을 로컬→클라우드로 바꿔도 이 **JSON 계약은 불변**이라 엔진은 안 건드려도 됨.

## 실행 방법

### 웹앱 (엔진 포함, 서버 불필요)
```bash
# 그냥 index.html 을 브라우저로 열거나, 로컬 서버로:
cd saju && python3 -m http.server 8000
# → http://localhost:8000
```
방문자 브라우저에서 100% 계산된다. 외부 API 호출 없음(오프라인·무료·무제한).

### Python 엔진 (검증·백엔드)
```bash
cd engine
python3 demo.py 1998 6 8 10 0 F          # 콘솔 렌더
python3 demo.py 1998 6 8 10 0 F --json   # 구조화 JSON (LLM 계약 형태)
python3 tests/test_fixture_19980608.py   # 픽스처 검증 (31/31 통과해야 정상)
```
표준 라이브러리만 사용(`math`, `datetime`). 외부 패키지 불필요.

### JS 엔진 단독 (node 교차검증)
```bash
node -e 'const S=require("./js/saju-engine.js"); console.log(S.buildChart(1998,6,8,10,0,"F").strength);'
```

## ⚠️ 엔진 동기화 (JS ↔ Python)

**가장 중요한 유지보수 규칙.** 같은 로직이 두 언어로 존재하므로 **한쪽만 고치면 값이 갈라진다.**
(실제로 과거에 강약 공식을 JS만 고쳐서 지수가 56↔51로 어긋난 적 있음 → 2026-07 동기화 복구.)

강약/용신처럼 계수가 있는 함수를 고칠 때 **반드시 두 파일을 같이** 고칠 것:

| 로직 | JS | Python |
|---|---|---|
| 강약 가중치 `POSW`/`POS_WEIGHT` | `js/saju-engine.js` `strength()` | `engine/saju/strength.py` |
| 강약 지수 공식 (0.45·세력 + 0.22·통근 + 0.33·득) | 〃 | 〃 |
| 밴드 경계 `CUTS`/`BAND_THRESHOLDS` `[18,30,41,48,57,68,80]` | 〃 | 〃 |
| 조후/억부 용신 | `johu()`/`eokbu()` | `johu_yongshin()`/`eokbu_yongshin()` |

**고친 뒤 교차검증** (둘의 `strength.index`가 같아야 함):
```bash
cd engine && python3 tests/test_fixture_19980608.py   # 지수 확인
cd ..     && node -e 'console.log(require("./js/saju-engine.js").buildChart(1998,6,8,10,0,"F").strength.index)'
```

## 보정 포인트 (레퍼런스 사이트에 맞출 때)

- **밴드 경계** `CUTS`/`BAND_THRESHOLDS` : 8단계 신강약 컷.

- **인구분포 %** : 재현 불가(사이트 자체 통계). 필요 시 자체 통계 DB 구축.

## 기술 노트

- **24절기** = 로컬 천문계산(Meeus 태양황경, `solarterms.py` / JS `sunLon`). 절입 시각을 분 단위로
  산출 → 년주(입춘)·월주(12절)·대운수를 좌우. **전 연도 커버, 오프라인·무제한.**
  정밀도: 절기 **날짜(일)는 정확**, 시각은 알려진 값 대비 **~수 분** 오차(저정밀 근사).
  초정밀이 필요하면 태양황경 함수만 VSOP87로 교체(구조 동일).
- **KASI 음양력 API의 위치** : 세차(년주)·월건(월주)은 음력 기준이라 절기 경계에서 명리와 어긋남.
  **일주(일진)와 음↔양력 변환만** 신뢰. → 이 엔진은 KASI에 의존하지 않고 자체 계산하며,
  KASI는 교차검증용. (자세한 조사 결과는 `engine/README.md`)
- **음↔양력** = `js/lunar-table.json`(1901~2099), KASI 교차검증 완료.
