# 사주 만세력 엔진 (saju-engine)

생년월일시 + 성별 → **화면 3장(원국·신강약/용신·대운/세운)의 모든 필드**를 산출하는
**결정론적 만세력 엔진**. 상용화 대비 설계.

```python
from saju.chart import build_chart
chart = build_chart(1998, 6, 8, 10, 0, gender="F")   # 양력, KST
```
```bash
python3 demo.py 1998 6 8 10 0 F          # 콘솔 렌더
python3 demo.py 1998 6 8 10 0 F --json   # 구조화 JSON(LLM 계약)
python3 tests/test_fixture_19980608.py   # 픽스처 검증
```

## ⚠️ 핵심 설계 원칙 — 계산과 해석의 분리

```
[이 엔진]  결정론적 계산 (오차 0)          [LLM 레이어]  서술만
 8글자·십성·지장간·12운성·12신살    ──JSON──▶   성향·운세 해석
 신강약·용신·대운·세운                        (지금:로컬LLM → 나중:클라우드 API)
```

**만세력은 절대 LLM에게 계산시키지 않는다.** LLM은 `build_chart()`가 반환한 JSON을
*읽어서 서술*만 한다. 해석 LLM을 로컬 → 클라우드 API로 교체해도 이 계약(JSON)은 불변.

## 24절기 (정확도의 핵심)

- **로컬 천문계산**(Meeus 태양황경, `solarterms.py`)으로 절입 시각을 분 단위 산출.
  → 년주(입춘)·월주(12절)·대운수를 좌우. 오프라인·무제한.
- 정밀도: 절기 **날짜(일)는 정확**, 시각은 알려진 값 대비 **~수 분** 오차(Meeus 저정밀).
  경계 '분' 단위 초정밀이 필요하면 `_sun_apparent_longitude()`를 VSOP87로 교체(구조 동일).
- **KASI 절기 API**(`get24DivisionsInfo`)는 현재 키 미등록(403). 특일정보 활용신청 후
  `kasi_crosscheck()` 훅으로 *일 단위 교차검증* 연결 가능(정확도 병목 아님).

## KASI 음양력 API의 위치

- 검증 결과: KASI 음양력 API의 세차(년주)·월건(월주)은 **음력 기준**이라 절기 경계에서
  명리 간지와 어긋남. **일주(일진)와 음↔양력 변환만** 신뢰 가능.
- 따라서 이 엔진은 KASI에 의존하지 않고 자체 계산하며, KASI는 **일주/변환 교차검증용**.

## 파일 구조

| 파일 | 역할 |
|---|---|
| `saju/constants.py` | 고정표(천간·지지·오행·지장간·십성·12운성·12신살). 계산 로직 없음. |
| `saju/solarterms.py` | 24절기 천문계산 + KASI 교차검증 훅 |
| `saju/pillars.py` | 4기둥(년=입춘/월=절기+월두법/일=일진/시=시두법) |
| `saju/derive.py` | 십성·지장간·12운성·12신살 파생 |
| `saju/strength.py` | 신강약(득령지시세·8밴드) + 용신(조후/억부) |
| `saju/luck.py` | 대운(대운수·순역·전개) + 세운 |
| `saju/chart.py` | 통합 → LLM 계약 JSON |

## 보정 포인트 (레퍼런스 사이트에 맞출 때)

- `strength.BAND_THRESHOLDS` : 8단계 밴드 경계
- `strength.POS_WEIGHT / ROOT_STRENGTH` : 강약 가중치
- `strength.johu_yongshin` : 조후는 간이표 → 窮通寶鑑 전체표로 상향 가능
- `pillars` late_zi 옵션 : 야자시(23시) 일주 경계 처리 유파 선택
